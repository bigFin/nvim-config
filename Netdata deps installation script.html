Here’s a Bash script to install the required dependencies, set up the Netdata source tree, build, and install Netdata. It assumes the script is being run on a Linux system with sudo access.

bash
#!/bin/bash

# Exit script on error
set -e

# Function to check if a command exists
command_exists() {
    command -v "$1" &>/dev/null
}

# Update and install dependencies
echo "Installing dependencies..."
sudo apt update

# Install minimum required dependencies
sudo apt install -y \
    uuid-dev \
    libuv1-dev \
    zlib1g-dev \
    cmake \
    gcc \
    make \
    git \
    ninja-build

# Install additional dependencies
sudo apt install -y \
    libyaml-dev \
    libjson-c-dev \
    libssl-dev \
    liblz4-dev \
    protobuf-compiler

# Install Go for Go collectors
if ! command_exists go; then
    echo "Installing Go..."
    GO_VERSION="1.21.0"
    wget "https://go.dev/dl/go${GO_VERSION}.linux-amd64.tar.gz"
    sudo tar -C /usr/local -xzf "go${GO_VERSION}.linux-amd64.tar.gz"
    rm "go${GO_VERSION}.linux-amd64.tar.gz"
    export PATH="/usr/local/go/bin:$PATH"
    echo "export PATH=/usr/local/go/bin:\$PATH" >> ~/.bashrc
    source ~/.bashrc
fi

# Clone the Netdata repository
echo "Cloning the Netdata repository..."
if [ ! -d "netdata" ]; then
    git clone https://github.com/netdata/netdata.git
fi

cd netdata

# Update Git submodules
echo "Updating Git submodules..."
git submodule update --init --recursive

# Build and install Netdata
echo "Configuring and building Netdata..."
cmake -S . -B build -G Ninja
cmake --build build --parallel
sudo cmake --install build

echo "Netdata installation completed successfully!"

Notes:

  1. Dependencies: This script assumes apt for package management. If you’re on a different system (e.g., yum for RHEL/CentOS), the package names may differ.
  2. Go Installation: If go is already installed, it skips installation. Adjust the GO_VERSION if a newer version is required.
  3. CMake Generator: Uses Ninja for faster builds. If unavailable, remove the -G Ninja flag to use Make.
  4. Source Directory: The script clones the Netdata repo into the current directory if it doesn’t already exist. Adjust the directory or clone URL as needed.
  5. Run as Root: The script uses sudo to install dependencies and Netdata. Ensure the user running the script has sudo privileges.
